Aggregate the Mesa preflight fixes onto a single branch so a preflight image can carry all of them: the Rosetta zkApp account-update multiplicity fix (#19047), the Rosetta zkApp account-creation-fee operation (#19052), the archive hardfork toolbox `populate-genesis-accounts` command and its `convert-canonical` / `is-in-best-chain` polish (#19056, #19057, #19059), and the archive Docker image bundling `mina-<network>-config` so the toolbox can resolve a fork genesis ledger in place (#19060). Additionally, because the multiplicity fix expands a command's account-update array positionally, the creation fee is now attributed to only the first occurrence of the creating update in a block — the ledger charges it once however many times that update is applied.

Also fix `mina-missing-blocks-auditor` on hardfork archives: it compared the canonical chain's *length* against the tip *height*, which assumes genesis sits at height 1. An archive that starts at a fork block is shorter than its tip height by everything below the fork, so the auditor reported a chain-length error (exit code 4) on a perfectly healthy post-fork archive — e.g. 11713 blocks "expected: 308084" for an archive rooted at 296372. The expected length now accounts for the fork block height, which the tool already looks up. Behaviour on a genesis-at-1 archive is unchanged.
