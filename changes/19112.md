Add flags to generate heavy zkApp transactions with the zkapp_test_transaction tool

The `update-state` subcommand of the `zkapp_test_transaction` tool now accepts `--num-events`, `--num-actions` and `--elements-per` flags, which let it generate zkApp commands carrying large events and actions arrays. This is useful for load and memory testing of the archive node. Omitting the flags preserves the previous behaviour.
