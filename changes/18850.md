Fixes Pickles proof serialization for chunked proofs.

Previously, Pickles.Proof.Make(...).to_base64 serialized proofs through the stable Repr adapter. That adapter collapses prev_evals.evals.public_input from arrays to a single pair of field elements by taking index 0. This is fine for unchunked proofs, but it loses public-input evaluation data when a proof spans multiple chunks. As a result, a freshly generated chunked proof could verify, but the same proof failed after to_base64 / of_base64 roundtrip.

This PR adds a dedicated Base64_repr for proof base64 serialization that preserves the full Base.Wrap.t shape, including the full prev_evals public-input arrays required by chunked proofs. The existing stable/binable representation remains unchanged, and of_base64 includes a fallback path for decoding proofs serialized with the previous representation.

Regression coverage was added to chunked_circuits/chunks2.ml: the test now verifies both a direct two-chunk proof and the same proof after base64 serialization/deserialization. It also checks the recursive proof roundtrip path.

Validated with:

```
dune build src/mina/src/lib/crypto/pickles
dune exec src/mina/src/lib/crypto/pickles/test/chunked_circuits/chunks2.exe
```
