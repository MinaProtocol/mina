# PR #18926 Buildkite: remove Rosetta artifact variant (build rosetta from apps-only + config)

CI/Tooling: removes the monolithic `Artifacts.Type.Rosetta` Dhall variant. Rosetta is now built only from `RosettaAppsOnly` (the generic apps image) plus `RosettaConfig` (the network-config layer on top), mirroring the existing `DaemonAppsOnly` + `DaemonConfig` split. The `rosetta_<network>` debian name moves from `Rosetta` to `RosettaAppsOnly`, so the `mina-rosetta-<network>` deb keeps being built and installed into the apps image. Release jobs and the hardfork entrypoint now produce `RosettaAppsOnly` + `RosettaConfig` instead of `Rosetta`; the rosetta connectivity test depends on `RosettaConfig` (the configured image it pulls), and the rosetta integration test already relies on rosetta apps (installs the debs directly). Final published image names/tags (`mina-rosetta:<ver>-<network>` and `...-generic`) are unchanged, so `manager.sh` promotion and downstream consumers are unaffected.
