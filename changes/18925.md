The mina-dispatch hardfork dispatcher now sanitizes explicit config files in post-fork (mesa) mode: when `--config-file` or `MINA_CONFIG_FILE` contain pre-fork daemon hardfork stop fields (`daemon.slot_tx_end`, `daemon.slot_chain_end`, `daemon.hard_fork_genesis_slot_delta`), those fields are automatically removed from a temporary copy before launching the post-fork daemon. When `jq` is unavailable a warning is emitted instead. The implicit `<config_dir>/daemon.json` is also warned about when those fields are present, since appended runtime configs cannot unset fields that Mina already loaded from the implicit file.
