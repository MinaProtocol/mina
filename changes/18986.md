Make `runtime_genesis_ledger` load hardfork accounts verbatim by default, matching the daemon's `Account.Hardfork.migrate_to_mesa` behavior for a Mesa → Mesa fork. The tool previously applied the Berkeley → Mesa vesting re-base (`Account.slot_reduction_update`) whenever a hardfork slot was derived from `--prefork-genesis-config`, which re-based already-Mesa timed accounts and produced a genesis-ledger root that diverged from the daemon/automode root. The implicit hardfork-slot derivation and the now-purposeless `--prefork-genesis-config` flag are removed (release scripts updated accordingly). For forks that still need Berkeley → Mesa vesting re-basing, `runtime_genesis_ledger` now exposes an explicit `--hardfork-slot` flag to apply `Account.slot_reduction_update` with the user-provided slot. Fixes #18981.
