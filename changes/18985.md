Fixed archive block-insert failures for max-cost zkApps that overflowed the Postgres btree key limit on `zkapp_events.element_ids` and `zkapp_field_array.element_ids` (`index row size ... exceeds btree version 4 maximum 2704`). The unbounded `int[] NOT NULL UNIQUE` is replaced with a UNIQUE index over a fixed-width sha256 content hash (`zkapp_element_ids_hash(element_ids)`), restoring content-deduplication and indexed lookup without the overflow, for both tables. Includes the change in `upgrade_to_mesa.sql` (drop old constraint → deduplicate existing rows → add the hash index) for berkeley→mesa migrations, and a standalone `migrate_zkapp_element_ids_to_hash.sql` (dry-run by default, with integrity gates) for archive nodes already on mesa to apply the fix in place.
