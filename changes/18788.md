**Log rotation now uses timestamps:** Rotated log files are now named with ISO 8601 timestamps (e.g. `mina.log.2024-01-15T10:30:00Z`) instead of numeric indices (`mina.log.0`, `mina.log.1`). The timestamp reflects when the log file was first opened, so you can tell at a glance what time period each file covers. If multiple rotations happen within the same second, a sequence number is appended (e.g. `mina.log.2024-01-15T10:30:00Z.1`). The rotation limit (`--file-log-rotations`) still applies — oldest files beyond the limit are automatically cleaned up.
