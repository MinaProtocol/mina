Restructure the Mina Docker images into layered build stages

The mina-daemon, mina-archive, and mina-rosetta images are now assembled from shared, layered stages instead of monolithic Dockerfiles, and the common base-dependencies layer is published as a reusable mina-base image. A standard daemon image no longer bundles the pinned pre-fork genesis ledger by default — that is now an opt-in layer used only for hardfork builds — so a normal daemon image builds without any legacy packaging dependency. The behaviour of the running daemon, archive, and rosetta containers is unchanged.
