Fixed archive block inserts failing for zkApps that emit large event lists. The `zkapp_events.element_ids` column was `int[] NOT NULL UNIQUE`, and the UNIQUE constraint is backed by a Postgres btree; once a zkApp emitted enough events (~1030, a ~4128-byte array) the btree key exceeded Postgres' 2704-byte limit, index maintenance failed, and the block rolled back. The UNIQUE constraint and its index are dropped (deduplication of event lists is handled in code via SELECT-before-INSERT), and `upgrade_to_mesa.sql` drops them idempotently for existing databases.
