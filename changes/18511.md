The new formula for the `chain_id` can be computed from the config file (ie without
unpacking the ledger) as long as the config file declares the ledger hashes.
This is the case for all production environments. We use this value to nest
the `chain_state_locations` under the `chain_id` filepath, which allows us
to store multiple chain states in the same mina config dir.

Daemon's client side command `mina client chain-id-inputs` is removed.
