Refuse to overwrite a Docker image that already exists in the remote registry. `scripts/docker/build.sh` now checks the resolved tag with `docker manifest inspect` before building and exits with an error if it is already published, so a build cannot silently replace an image someone else depends on (for example a hardfork build clobbering a released devnet image). The check runs before the buildx invocation so it fails fast rather than after a full image build, and it only applies when the script would actually push (`--load-only` builds are unaffected). Setting `FORCE_DOCKER_OVERWRITE` to any non-empty value opts back in to overwriting; the hardfork package pipeline sets it, while regular release pipelines stay protected.
