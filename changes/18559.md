- Added a filesystem hint at `/etc/coda/build_config/PROFILE`.
  This file tells the node which configuration profile to load at startup.

- Added support for selecting the node configuration profile at runtime via the
  `MINA_PROFILE` environment variable. Supported values are `dev`, `devnet`,
  `lightnet`, and `mainnet`.

  `MINA_PROFILE` overrides the packaged profile hint and is intended for advanced
  operators who need to run a Mina binary with a different profile. Use this with
  care, and ensure the runtime configuration and genesis ledger match the
  selected profile.

- Debian packages now install the packaged profile hint through a profile package,
  such as `mina-devnet-profile` or `mina-mainnet-profile`. These packages are
  pulled in automatically as dependencies of the Mina daemon, archive, and
  Rosetta packages.

- Mesa packages use the devnet profile hint and depend on `mina-devnet-profile`.
